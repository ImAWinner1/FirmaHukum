# DESIGN.md

## 1. Purpose of This Document

This document defines the visual identity, user experience direction, layout principles, design system rules, and interface implementation standards for the website of a law firm.

The purpose of this file is to ensure that every screen, section, component, spacing decision, type treatment, color usage, and interaction pattern follows a consistent professional standard. This website must communicate credibility, authority, confidence, restraint, and clarity.

The design must feel appropriate for a legal services business. It should not feel playful, trendy in a disposable way, overly decorative, or visually noisy. The experience must be calm, premium, refined, and trustworthy.

This document serves as the design authority for the project. Any new UI or UX decision should be evaluated against the principles described here.

---

### 2.1 Brand Perception Goal

The website should immediately suggest:

- professionalism,
- legal credibility,
- reliability,
- seniority,
- restraint,
- attention to detail,
- strong client service,
- premium corporate quality.

### 2.2 Emotional Objective

The user should feel:

- safe,
- informed,
- respected,
- confident in the firm,
- motivated to contact the firm,
- reassured that the firm is competent and serious.

### 2.3 Design Character

The visual style should be:

- elegant,
- structured,
- formal,
- clean,
- premium,
- modern,
- timeless,
- minimal but not empty,

## 3. Core Design Principles

The entire interface must be built around the following principles.

### 3.1 Clarity First

Every page must communicate its purpose quickly. The user should always understand:

- where they are,
- what the firm offers,
- what action is available next.

### 3.2 Visual Hierarchy

Important content must stand out naturally through:

- typography,
- spacing,
- contrast,
- scale,
- section order,
- CTA prominence.

### 3.3 Breathing Room

Use generous whitespace. A law firm website should never feel crowded or cramped.

### 3.4 Balance and Restraint

The interface should feel composed. Avoid excessive decoration, random flourishes, or competing visual accents.

### 3.5 Consistency

Spacing, typography, colors, shadows, borders, and motion must remain consistent across the site.

### 3.6 Trust Through Simplicity

Trust is often strengthened by simplicity. The design should never feel chaotic or over-engineered.

### 3.7 Premium Execution

Premium quality comes from precision:

- alignment,
- proportions,
- line height,
- spacing rhythm,
- image treatment,
- button styling,
- hover states.

---

## 4. Overall UI Direction

### 4.1 Recommended Style

The site should follow a **premium corporate legal website** aesthetic.

The design language should resemble a high-end professional services website, not an e-commerce page, not a startup toy site, and not a portfolio filled with visual gimmicks.

### 4.2 What the Interface Should Avoid

Avoid:

- cartoonish elements,
- overly bright colors,
- random gradients,
- neon themes,
- excessive glassmorphism,
- heavy shadows,
- cluttered card layouts,
- crowded hero sections,
- decorative noise,
- fake statistics,
- “template-looking” visuals.

### 4.3 Visual Tone

The tone should be:

- serious,
- polished,
- polished but not cold,
- professional but still approachable,
- modern but rooted in trust.

## 5. Color System

Color should reinforce authority and readability. A law firm website should use a controlled palette with strong neutrals and a single elegant accent color.

### 5.1 Primary Palette Recommendation

Use a palette built around:

- deep navy / midnight blue,
- charcoal / near-black,
- warm gold / muted brass accent,
- white,
- soft gray neutrals.

### 5.2 Suggested Color Roles

#### Primary Background

- Very dark navy or near-black for premium sections
- White for content-heavy sections if contrast is needed

#### Primary Text

- Near-black on light backgrounds
- Soft white / off-white on dark backgrounds

#### Accent Color

- Muted gold, brass, or refined amber
- Use only for key highlights, icons, CTA emphasis, section markers, and subtle lines

#### Supporting Neutrals

- Cool gray for secondary text
- Light gray for borders and separators
- Dark slate for muted dark surfaces

### 5.3 Color Usage Rules

- Do not use accent color everywhere.
- Use the accent only where emphasis is truly needed.
- Keep saturation low.
- Use a limited number of colors so the site remains mature and refined.
- Contrast must always support readability.

### 5.4 Example Palette Direction

The exact final palette may be refined, but the visual direction should stay within this family:

- Background: deep navy / near-black
- Surface: dark slate / white
- Text: off-white / near-black
- Accent: gold / brass
- Border: muted gray

## 6. Typography System

Typography is one of the most important parts of a legal website. It must feel authoritative and highly readable.

### 6.1 Typeface Recommendation

Use a clean sans-serif family for the main system.

Recommended direction:

- **Primary font:** Inter, Plus Jakarta Sans, Geist, or a similarly modern professional sans-serif
- **Fallback stack:** system-ui, -apple-system, Segoe UI, sans-serif

### 6.2 Typography Goals

Typography must:

- improve clarity,
- support long-form reading,
- create a premium feel,
- maintain strong hierarchy,
- remain readable on all devices.

### 6.3 Type Scale Suggestion

Use a clear and restrained scale.

#### Headings

- Large hero headings
- Medium section headings
- Small card titles

#### Body Text

- Comfortable paragraph size
- High line-height
- Limited line width for readability

#### Supporting Text

- Smaller labels, captions, metadata, role labels, and helper text

### 6.4 Typographic Hierarchy Rules

- Headlines should be strong but not oversized.
- Body text must remain comfortable to read.
- Labels should be subtle but legible.
- Legal language must be readable without strain.

### 6.5 Line Height and Spacing

- Headings should have tight but not cramped line height.
- Body copy should have generous line height.
- Paragraph blocks should have enough spacing to avoid visual fatigue.

### 6.6 Typography Style Rules

- Use bold weights sparingly and purposefully.
- Avoid too many font weights.
- Avoid decorative fonts.
- Avoid scripts, serif display fonts, or overly stylized headings unless explicitly approved.

---

## 7. Spacing and Layout Rhythm

### 7.1 Spacing Philosophy

Spacing must be deliberate. The site should breathe.

### 7.2 Core Spacing Rules

- Use consistent section padding across the website.
- Use generous vertical spacing between major sections.
- Use smaller internal spacing inside cards and forms.
- Maintain a stable rhythm between headline, subheadline, and content.

### 7.3 Layout Density

The layout should be moderate-to-relaxed in density.

- Hero sections can be spacious.
- Content sections should remain structured.
- Dense blocks of text must be broken into readable subsections.

### 7.4 Grid Discipline

- Use a clean grid.
- Keep alignment consistent.
- Avoid asymmetry unless it has a clear design purpose.
- Use predictable column structures.

### 7.5 Section Separation

Separate major sections using:

- spacing,
- background shifts,
- subtle borders,
- content hierarchy,
- controlled visual contrast.

Do not separate sections using aggressive visual effects.

---

## 8. Layout Strategy

### 8.1 Page Structure Philosophy

The website should be easy to scan. The page order must feel logical:

1. Hero
2. Firm overview / credibility
3. Practice areas
4. Why choose us
5. Team profile
6. Process or consultation flow
7. FAQ
8. Contact
9. Footer

### 8.2 Desktop Layout

On desktop, the site should feel editorial and premium:

- wide margins,
- clear content blocks,
- thoughtful use of two-column layouts,
- strong visual hierarchy,
- stable reading width.

### 8.3 Tablet Layout

On tablet, the design should preserve hierarchy and avoid unnecessary compression. Components should stack gracefully.

### 8.4 Mobile Layout

On mobile, the site must remain elegant and easy to navigate:

- single-column stacking,
- concise copy blocks,
- clear CTA placement,
- readable typography,
- accessible touch targets,
- reduced visual clutter.

### 8.5 Container Width

Content width should be controlled to preserve readability. Long line lengths reduce trust and comfort. Use a maximum content width that supports premium editorial composition.

## 9. Section Design Rules

### 9.1 Hero Section

The hero section must immediately explain:

- who the firm is,
- what it offers,
- why it is credible,
- what the user should do next.

Hero requirements:

- strong headline,
- concise supporting paragraph,
- primary CTA,
- secondary CTA,
- optional hero visual or abstract legal composition,
- restrained premium styling.

### 9.2 Trust Section

A law firm site should quickly establish trust.
Possible trust-building elements include:

- years of experience,
- practice areas,
- client types,
- awards or memberships if approved,
- office location,
- consultation promise,
- confidentiality emphasis.

### 9.3 Practice Areas Section

Practice areas should be presented clearly and elegantly.
Each service card should contain:

- title,
- short description,
- subtle icon,
- optional hover refinement.

### 9.4 Team Section

Lawyer profiles should feel authoritative, formal, and polished.
Requirements:

- professional photo presentation,
- clear name,
- legal title,
- specialization,
- short bio,
- consistent card style.

### 9.5 FAQ Section

FAQ should use a clean accordion design.
Questions must be short, practical, and client-oriented.

### 9.6 Contact Section

Contact should feel approachable and frictionless.
Requirements:

- address,
- phone,
- email,
- office hours,
- map embed or map link,
- contact form,
- direct CTA buttons.

## 10. Component Design System

### 10.1 Buttons

Buttons should be simple, clear, and premium.

#### Primary Button

- filled accent color,
- strong contrast,
- clear hover state,
- reserved for main CTA.

#### Secondary Button

- outline or subtle surface style,
- used for less critical actions.

#### Button Rules

- never overload with multiple competing styles,
- keep padding consistent,
- ensure all buttons are easy to tap on mobile.

### 10.2 Cards

Cards should be clean and elegant.

Card rules:

- subtle border or surface contrast,
- consistent corner radius,
- controlled shadow usage,
- enough padding,
- clear title hierarchy,
- no cluttered internal layout.

### 10.3 Forms

Forms must look professional, calm, and easy to use.

Form rules:

- labels should be visible,
- inputs should be spaced properly,
- error states must be clear,
- success states must be reassuring,
- submit buttons should be obvious.

### 10.4 Navigation

Navigation should be simple and stable.

Navigation rules:

- minimal top-level items,
- sticky behavior may be used if subtle,
- active section indication may be added,
- mobile menu must be easy to use.

### 10.5 Footer

The footer should feel structured and complete.
Include:

- firm name,
- copyright,
- contact details,
- quick links,
- legal or policy links if required.

## 12. Motion Design Rules

### 12.1 Motion Philosophy

Motion must support polish and clarity, not spectacle.

### 12.2 Motion Types Allowed

Use only restrained motion such as:

- fade in,
- slight slide in,
- gentle scale,
- subtle hover transitions,
- smooth section reveal.

### 12.3 Motion Limits

Avoid:

- bouncing animations,
- flashy transitions,
- spinning elements,
- excessive parallax,
- large movement distances,
- distracting repeated motion.

### 12.4 Motion Timing

Motion should feel:

- smooth,
- controlled,
- professional,
- consistent.

## 18. Design System Organization

The implementation should remain easy to maintain by keeping design assets and UI logic organized.

### 18.1 Recommended Design-Focused Folder Structure

A design-oriented Next.js project may be organized like this:

```text
src/
├── app/
├── components/
│   ├── layout/
│   ├── sections/
│   ├── ui/
│   └── shared/
├── features/
│   ├── contact/
│   ├── lawyers/
│   ├── practice-areas/
│   └── faq/
├── constants/
├── lib/
├── hooks/
├── types/
├── styles/
├── public/
│   ├── images/
│   ├── icons/
│   └── logos/
└── data/
```

### 18.2 Folder Meaning

- `components/ui`: foundational reusable components
- `components/sections`: page sections composed from smaller pieces
- `components/layout`: header, footer, shell, navigation
- `features`: business-feature modules grouped by domain
- `constants`: stable values such as nav items, legal practice names, or social links
- `data`: content collections or mock content used for structured rendering
- `public/images`: optimized static assets

### 18.3 Organization Rules

- Keep layout, section, and shared UI logic separated.
- Do not store unrelated elements in the same folder.
- Group feature-specific components near their feature domain when the project grows.

## 20. Design Tokens and Consistency

### 20.1 Design Tokens

Maintain consistent values for:

- spacing,
- color,
- radius,
- shadow,
- typography size,
- motion duration,
- border width.

### 20.2 Token Discipline

Do not use random values when the same visual relationship is used repeatedly.

### 20.3 System Thinking

Treat the UI as a system, not as isolated screens.

---

## 21. Proportions and Visual Refinement

### 21.1 Proportion Control

The layout should feel balanced at every width.

### 21.2 Border Radius

Use a consistent and restrained radius scale.

### 21.3 Shadow Usage

Shadows should be subtle and functional, not dramatic.

### 21.4 Border Usage

Borders should support separation without heavy visual weight.

---

## 22. What the Design Must Not Become

The design must never become:

- generic template sludge,
- visually noisy,
- over-animated,
- inconsistent,
- childish,
- overly decorative,
- difficult to scan,
- difficult to navigate,
- too dark to read,
- too bright to feel premium.

---

## 23. Final Design Quality Bar

A design implementation is acceptable only if it is:

- visually coherent,
- elegant,
- professional,
- trustworthy,
- responsive,
- accessible,
- consistent,
- restrained,
- premium,
- suitable for a real law firm.

The final result should feel like a serious legal business website created with care, precision, and strong product judgment.

---

## 24. Closing Principle

The website must never look like a generic theme with a logo placed on top. It should feel intentionally designed for the legal profession. Every visual choice must support authority, clarity, and trust.

Design quality is not measured by how many effects are added. It is measured by how convincingly the interface communicates professionalism and confidence.
